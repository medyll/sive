# Story S16-05 — Unit tests — sharing & RBAC

**Epic:** QA
**Sprint:** Sprint 16
**Points:** 3
**Priority:** Must

## User Story

As a **developer**, I want comprehensive unit tests for the sharing data model and RBAC utilities, so that regressions are caught automatically.

## Context

Unit tests for all server-side logic added in S16-01 and S16-03.

## Acceptance Criteria

```gherkin
Given the document_shares schema
When a share is created
Then it is retrievable by getDocumentRole

Given a user with "viewer" role
When requireDocumentRole is called with minimumRole "editor"
Then it throws a 403 error

Given a user with "owner" role
When requireDocumentRole is called with any minimum role
Then it succeeds

Given no share record exists for a user+document
When getDocumentRole is called
Then it returns null
```

## Technical Notes

- Test file: `src/lib/server/rbac.spec.ts`
