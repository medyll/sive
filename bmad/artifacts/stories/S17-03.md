# Story S17-03 — Show shared documents in document list

**Epic:** Collaboration
**Sprint:** Sprint 17
**Points:** 3
**Priority:** Should

## User Story

As a **collaborator**, I want to see documents that have been shared with me in my document list, so that I can access them alongside my own documents.

## Context

Sprint 16 added document sharing data model and RBAC. Currently the document list (`+page.server.ts` load) only returns documents owned by the user (`user_id = userId`). Shared documents (editor/viewer role) are not shown.

## Acceptance Criteria

```gherkin
Given user A shares document X with user B as editor
When user B loads the app
Then document X appears in user B's document list

Given user B is a viewer of document Y
When user B loads the document list
Then document Y appears with a "Shared" indicator

Given user A owns document Z (not shared)
When user A loads the document list
Then document Z appears as normal (no shared indicator)
```

## Technical Notes

- Update `load` in `src/routes/app/+page.server.ts`:
  - Query `document_shares` for the current user's `user_id` to find all documents they have access to
  - Fetch those documents from the `documents` table
  - Merge with own documents, deduplicate, sort by `updated_at` descending
  - Add `role` field to each returned document item
- Update `DocumentList.svelte`: show a "Shared" badge next to documents where `role !== 'owner'`
- No change to existing mock stub behavior

## Out of Scope

- Filtering shared vs own documents
- Leaving a shared document

## Dependencies

- S16-01 (document_shares schema)
- S16-03 (getDocumentRole utility)
