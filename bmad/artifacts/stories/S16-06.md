# Story S16-06 — E2E tests — sharing flow

**Epic:** QA
**Sprint:** Sprint 16
**Points:** 3
**Priority:** Must

## User Story

As a **developer**, I want end-to-end tests covering the complete document sharing flow, so that the feature works correctly from the user's perspective in a real browser.

## Context

E2E tests using Playwright covering invite, role enforcement, and collaborator management.

## Acceptance Criteria

```gherkin
Given two registered users (owner and invitee)
When the owner shares a document with the invitee as "editor"
Then the invitee can open and edit the document

Given the invitee has "viewer" role
When the invitee attempts to save edits
Then the operation is rejected and an error toast appears

Given the owner opens the Share modal
When they change the invitee's role to "viewer"
Then the invitee loses edit privileges on the next action

Given the owner removes a collaborator
When the collaborator tries to access the document
Then they are redirected with a 403 message
```

## Technical Notes

- Test file: `e2e/sharing-flow.spec.ts`
- Use two browser contexts to simulate two users (Playwright multi-context)
- Extend existing Playwright fixtures to support multi-user sessions
- Tests must clean up created shares after each test
- Target: 8+ E2E test cases

## Out of Scope

- Performance testing of sharing under load
