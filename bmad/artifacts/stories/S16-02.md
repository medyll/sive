# Story S16-02 — Share invitation UI

**Epic:** Collaboration
**Sprint:** Sprint 16
**Points:** 3
**Priority:** Must

## User Story

As a **document owner**, I want to open an invitation dialog and search for users by email, so that I can invite them to collaborate on my document.

## Context

With the data model in place (S16-01), the owner needs a UI to manage invitations without leaving the editor.

## Acceptance Criteria

```gherkin
Given I am the document owner
When I click the "Share" button in the editor toolbar
Then a Share modal opens with a user-search input and current collaborators list

Given I type an email in the search input
When a matching registered user is found
Then their name and avatar appear as a suggestion

Given I select a user and choose a role (editor or viewer)
When I click "Invite"
Then the share record is created and the user appears in the collaborators list

Given I am not the document owner
When I view the editor toolbar
Then the Share button is disabled or hidden
```

## Technical Notes

- New component: `src/lib/elements/ShareModal.svelte`
- Server action: `POST /api/shares` — body: `{ documentId, targetEmail, role }`
- Server action: `GET /api/shares?documentId=` — returns list of shares with user info
- User search: `GET /api/users/search?q=<email>` — returns matching users (email prefix match)
- Role selector: dropdown with "Editor" / "Viewer" options
- Wire Share button into the editor toolbar (already contains toolbar items)

## Out of Scope

- Bulk invite
- Share via link
- Revoke / update role (handled in S16-04)

## Dependencies

- S16-01 (schema and `getDocumentRole`)
