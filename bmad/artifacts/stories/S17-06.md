# Story S17-06 — E2E tests — OAuth flow + shared docs UI

**Epic:** QA
**Sprint:** Sprint 17
**Points:** 3
**Priority:** Must

## User Story

As a **developer**, I want E2E tests for the OAuth login page and shared document display, so that UI regressions are caught automatically.

## Acceptance Criteria

```gherkin
Given the auth page loads in mock mode
When I view the page
Then the GitHub button is not visible (hasGithub: false in mock)

Given the app page loads with shared documents
When shared docs are in the list
Then a "Shared" badge is visible next to shared documents

Given the profile page loads
When I am authenticated
Then the display name input is visible
```

- [ ] 6+ new E2E tests passing
