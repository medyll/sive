# Story S16-03 — Role-based access control enforcement

**Epic:** Collaboration
**Sprint:** Sprint 16
**Points:** 3
**Priority:** Must

## User Story

As a **platform**, I want to enforce document access roles on every server-side operation, so that unauthorized users cannot read or modify documents they haven't been invited to.

## Context

Currently document routes are protected only by authentication (is the user logged in?). This story adds a second layer: is the user authorized for this specific document, and with what role?

## Acceptance Criteria

```gherkin
Given a viewer-role user is viewing a document
When they attempt to save changes via the document API
Then the server returns 403 Forbidden

Given an editor-role user edits a document
When they save changes
Then the change is persisted and the server returns 200

Given an owner attempts to delete a document
When the delete endpoint is called
Then only owners are permitted (editors and viewers receive 403)

Given an unauthenticated request reaches a document API endpoint
When the request is processed
Then the server returns 401 Unauthorized
```

## Technical Notes

- Create a SvelteKit server hook / layout server helper: `requireDocumentRole(event, documentId, minimumRole)`
  - `minimumRole` order: owner > editor > viewer
  - Throws `error(403)` if the user's role is below the minimum
- Apply `requireDocumentRole` to:
  - `GET /api/documents/[id]` — minimum: viewer
  - `PATCH /api/documents/[id]` — minimum: editor
  - `DELETE /api/documents/[id]` — minimum: owner
  - WebSocket join handler — minimum: viewer (update `src/lib/server/websocket/server.ts`)
- Add RBAC utility module: `src/lib/server/rbac.ts`
- All existing document API routes must be updated to call this utility

## Out of Scope

- Admin / super-admin roles
- Per-section permissions
