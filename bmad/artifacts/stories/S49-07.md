# S49-07 — redesign-documentlist-css

Summary

Redesign the DocumentList component's styling to improve visual hierarchy, accessibility and consistency with the project design system. Prefer conversion to Tailwind utility classes to align with repo patterns and make future maintenance easier. If Tailwind conversion is infeasible for any UI piece, modernize the scoped CSS and surface design tokens via CSS variables.

Goals

- Convert DocumentList.svelte to use Tailwind utility classes where practical.
- Replace fragile hard-coded colors with CSS variables (or Tailwind tokens).
- Improve spacing, focus states, and contrast for a11y.
- Keep existing behavior (bulk select, tag editing, context menu) intact.
- Update unit tests and Playwright snapshots affected by visual changes.

Acceptance criteria

- DocumentList.svelte renders correctly across desktop and narrow widths.
- Keyboard navigation, focus ring, and aria attributes remain functional.
- Unit tests that cover DocumentList updated and passing locally.
- A short PR description explaining the stylistic decisions and any trade-offs.

Assumptions

- Tailwind is available and already used elsewhere in the project.
- No schema or API changes are required.

Notes

- Recommended reviewer: frontend designer / Svelte developer.
- Timebox: small polish + tests (1-2 work days estimated).
