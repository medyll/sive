# Story S91-05 — CSS Migration to @medyll/css-base

**Sprint:** 91 — Post-Beta Cleanup  
**Status:** in_progress  
**Priority:** Medium  
**Effort:** 3-5 jours  
**Progress:** 60%

---

## Contexte

Le projet utilise `@medyll/css-base` (installé v0.1.4) mais ne l'exploite pas correctement :

- `layout.css` définissait des couleurs en dur au lieu d'utiliser les tokens
- 1082 occurrences de couleurs hardcoded dans les composants `.svelte`
- Tailwind v4 importé mais css-base sous-utilisé
- Dark mode géré manuellement au lieu de `light-dark()`

**Story liée :** S91-01 (fix-beta-feedback-issues) — cette migration CSS fait partie des améliorations de qualité.

---

## Objectif

Migrer progressivement le codebase pour utiliser correctement `@medyll/css-base` :

1. ✅ **layout.css refactoré** — utilise `@import '@medyll/css-base'` et les tokens
2. ✅ **+page.svelte migré** — toutes les couleurs hardcoded remplacées par des tokens
3. ✅ **TabBar.svelte migré** — tokens css-base utilisés
4. ✅ **ChatBar.svelte migré** — tokens css-base utilisés
5. ✅ **AIPanel.svelte migré** — tokens css-base utilisés
6. ⏳ **Composants `.svelte` restants** — remplacer les couleurs hardcoded par des tokens
7. ⏳ **Classes utilitaires** — utiliser `flex`, `gap-md`, `p-lg`, etc. au lieu de CSS custom
8. ⏳ **data-* attributes** — utiliser `data-elevation`, `data-pad`, `data-text` quand pertinent

## Notes techniques

### Patterns de migration

**Avant :**
```css
.sidebar {
  background: #f9fafb;
  border-right: 1px solid #e5e7eb;
  padding: 16px;
}
```

**Après :**
```css
.sidebar {
  background: var(--color-surface);
  border-right: 1px solid var(--color-border);
  padding: var(--pad-md);
}
```

**Ou avec utilitaires css-base :**
```html
<aside class="bg-surface border-r p-md">...</aside>
```

### data-* attributes (Chrome 139+)

```html
<div data-elevation="md" data-pad="lg" data-radius="md">
  Card with shadow, padding, rounded corners
</div>
```
