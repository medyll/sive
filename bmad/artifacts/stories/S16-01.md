# Story S16-01 — Document sharing data model

**Epic:** Collaboration
**Sprint:** Sprint 16
**Points:** 3
**Priority:** Must

## User Story

As a **document owner**, I want to share my document with other users and assign them roles, so that they can collaborate with appropriate permissions.

## Context

The real-time WebSocket collaboration is in place (Sprints 14–15). However, there is no persistent sharing model — any user can join a document URL without restriction. This story adds the DB schema and server-side model to manage document membership.

## Acceptance Criteria

```gherkin
Given a document exists
When its owner adds a share record for another user with role "editor"
Then the share record is persisted in the database with user_id, document_id, role, and created_at

Given a share record exists
When the server checks access for the shared user
Then the server returns the correct role ("owner" | "editor" | "viewer")

Given a non-invited user attempts to access a shared document
When the server evaluates their session
Then access is denied (403)
```

## Technical Notes

- Add `document_shares` table to Drizzle schema (`src/lib/server/db/schema.ts`):
  - `id` (uuid PK)
  - `document_id` (FK → documents.id)
  - `user_id` (FK → users.id)
  - `role` (text: "owner" | "editor" | "viewer")
  - `created_at` (timestamp ms)
- Add unique constraint on (document_id, user_id)
- On document creation, auto-insert a "owner" share record for the creating user
- Add server utility: `getDocumentRole(userId, documentId): Promise<Role | null>`
- Run `pnpm db:push` to apply schema (or generate migration)

## Out of Scope

- Email invitation flow
- OAuth-based sharing
- Share link (public URL)
