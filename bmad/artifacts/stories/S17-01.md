## Technical Notes

### Fix 1 — EditorToolbar.spec.ts:52 (`fireEvent is not defined`)
- `fireEvent` from `@testing-library/dom` is used but never imported
- Fix: replace `await fireEvent.click(focusButton)` with `focusButton.click()`

### Fix 2 — EditorToolbar.spec.ts:94 (`$set` removed in Svelte 5)
- `component.$set()` is Svelte 4 API, removed in Svelte 5
- Fix: remove the `component.$set()` call; the test only verifies the presence list element exists, which doesn't require prop update

### Fix 3 — HardenModal.svelte (Cancel button wiring)
- Cancel button calls `onclick={onCancel}` directly, bypassing `handleCancel`
- `handleCancel` is the one that dispatches `harden-cancel` on `rootEl`
- Fix: change Cancel button `onclick` from `onCancel` to `handleCancel`

### Fix 4 — RemoteCursor.svelte (stale cursor in DOM)
- `positionedCursors` filters by `isVisible` but NOT by recency
- Test expects cursor with `lastUpdated: 6s ago` to not appear in DOM
- Fix: add recency filter to `positionedCursors` computed value

### Fix 5 — performance.spec.ts:257 (timeout)
- Test loops for exactly 5000ms; vitest default timeout is also 5000ms
- Fix: add `{ timeout: 10000 }` as third argument to `it()`
