# Story S16-04 — Collaborators panel UI

**Epic:** Collaboration
**Sprint:** Sprint 16
**Points:** 3
**Priority:** Should

## User Story

As a **document owner**, I want to view and manage all collaborators from within the editor, so that I can update roles or remove access without navigating away.

## Context

After inviting users (S16-02), the owner needs a management panel to view active shares, change roles, and revoke access.

## Acceptance Criteria

```gherkin
Given I open the Share modal
When there are existing collaborators
Then I see a list of collaborators with their names, emails, and current roles

Given I click the role dropdown next to a collaborator
When I select a new role
Then the role is updated immediately and persisted

Given I click "Remove" next to a collaborator
When confirmed
Then the share record is deleted and the user loses access

Given I am an editor (not owner)
When I open the Share modal
Then I see a read-only collaborator list with no management controls
```

## Technical Notes

- Extend `ShareModal.svelte` with a collaborators list section
- Server action: `PATCH /api/shares/[shareId]` — body: `{ role }`
- Server action: `DELETE /api/shares/[shareId]`
- Optimistic UI update on role change (revert on error)
- Confirmation dialog before remove (use existing toast/modal pattern)

## Out of Scope

- Transfer ownership
- Bulk remove

## Dependencies

- S16-01 (schema)
- S16-02 (ShareModal base component)
- S16-03 (RBAC enforcement — ensures only owner can call PATCH/DELETE)
